4. Description of the system architecture

	A distributed filesystem-based model where three independent data providers are linked through a separate federated hub.

5. Explaination of the role of the metadata catalog

	It aggregates metadata descriptions to enable centralized discovery while the actual datasets remain stored locally at the providers.

6. Discussion questions
	
	1. Which components store the actual data?
	The data is stored only at the independent provider nodes: Satellite Provider A, Satellite Provider B, and the Ground Station Provider.	

	2. Which component enables dataset discovery?
	The federated metadata catalog enables discovery by storing and displaying metadata records.

	3. What advantages does the fedarated architecture provide?
	It allows for data collaboration without the need for centralizing or duplicating large datasets. It also ensures that providers maintain control over their own data.

	4. What limitations might such a system have? 
	Limitations include the need for manual updates to keep the catalog synchronized and the requirement for all participants to follow standardized metadata and access interfaces.
